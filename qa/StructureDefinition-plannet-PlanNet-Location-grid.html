NameCard.TypeConstraints and Usage
  Location 0..*dom-2: If the resource is contained in another resource, it SHALL NOT contain nested Resources
dom-4: If a resource is contained in another resource, it SHALL NOT have a meta.versionId or a meta.lastUpdated
dom-3: If the resource is contained in another resource, it SHALL be referred to from elsewhere in the resource or SHALL refer to the containing resource
dom-6: A resource should have narrative for robust management
dom-5: If a resource is contained in another resource, it SHALL NOT have a security label
Definition:
A PlanNet-Location is the physical place where healthcare services are provided, practitioners are employed, organizations are based, etc. Locations can range in scope from a room in a building to a geographic region/area.

  status 0..1codeBinding: LocationStatus (required)
Definition:
The status property covers the general availability of the resource, not the current value which may be covered by the operationStatus, or by a schedule/slots if they are configured for the location.

  name 1..1stringDefinition:
Name of the location as used by humans. Does not need to be unique.


Comments:
If the name of a location changes, consider putting the old name in the alias column so that it can still be located through searches.

  telecom 0..*ContactPointDefinition:
The contact details of communication devices available at the location. This can include phone numbers, fax numbers, mobile numbers, email addresses and web sites.

  address 0..1AddressDefinition:
Physical location.


Comments:
Additional addresses should be recorded using another instance of the Location resource, or via the Organization.

  line 0..*stringDefinition:
This component contains the house number, apartment number, street name, street direction, P.O. Box number, delivery hints, and similar address information.

  city 0..1stringDefinition:
The name of the city, town, suburb, village or other community or delivery center.

  state 0..1stringBinding: USPS Two Letter Alphabetic Codes (extensible)
Definition:
Sub-unit of a country with limited sovereignty in a federally organized country. A code may be used if codes are in common use (e.g. US 2 letter state codes).

  postalCode 0..1stringDefinition:
A postal code designating a region defined by the postal service.

  managingOrganization 0..1Reference(US Core Organization Profile)Definition:
The organization responsible for the provisioning and upkeep of the location.


Comments:
This can also be used as the part of the organization hierarchy where this location provides services. These services can be defined through the HealthcareService resource.
